# Project story

KinasePhos 4.0 was rebuilt from an architecture contract and a set of historically named data assets,
with no inherited code, checkpoint, or trusted log. The project emphasizes the parts that determine
whether a scientific ML result is credible: provenance, coordinate validation, duplicate evidence,
homology-aware splits, explicit task masks, reproducible caches, and honest metric status.

The supplied presentation is retained as project narrative, not as executable truth.
Its historical metrics remain unverified, its protein-generation section remains roadmap-only, and
its legacy slide-six role split is superseded by the exact architecture specification.
